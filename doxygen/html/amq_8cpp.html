Checks for the presence of a message from ActiveMQ and, if a message was received, returns it.

Parameters
[in]ms_waitnumber of milliseconds to wait for a message. if 0 then it will not wait.
[out]ierr0 indicates success (note a successful call can return no message) 1 indicates an internal error has occurred.
Returns
on successful exit either is NULL to indicate there are no messages or contains a null terminated char * shakeAlert decision module message
