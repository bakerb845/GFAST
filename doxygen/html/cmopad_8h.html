int cmopad_findFaultPlanes ( int  iverb,
struct cmopad_struct *  src 
)

Sets the two angle-triples, describing the faultplanes of the Double Couple, defined by the eigenvectors P and T of the moment tensor object.

Define a reference Double Couple with strike = dip = slip-rake = 0, the moment tensor object's DC is transformed (rotated) w.r.t. this orientation. The respective rotation matrix yields the first fault plane angles as the Euler angles. After flipping the first reference plane by multiplying the appropriate flip-matrix, one gets the second fault plane's geometry.

All output angles are in degree

Parameters
[in]iverbcontrols verbosity for debugging. 0 should be quiet.
[in,out]srcon input holds plot_clr_order on output holds the strike, dips, and rakes describing fault planes fp1 and fp2
Returns
0 indicates success

int cmopad_MT2PrincipalAxisSystem ( int  iverb,
struct cmopad_struct *  src 
)

Read in Matrix M and set up eigenvalues (EW) and eigenvectors (EV) for setting up the principal axis system.

The internal convention is the 'HNS'-system: H is the eigenvector for the smallest absolute eigenvalue, S is the eigenvector for the largest absolute eigenvalue, N is the null axis.

Naming due to the geometry: a CLVD is Symmetric to the S-axis, Null-axis is common sense, and the third (auxiliary) axis Helps to construct the RA3.

Additionally builds matrix for basis transformation back to NED system.

The eigensystem setup defines the colouring order for a later plotting in the BeachBall class. This order is set by the `_plot_clr_order' attribute.

Parameters
[in]iverbcontrols verbosity. 0 is quiet.
[in,out]srcon input holds the input moment tensor on output has the eigenvalues and fault planes

Brings the provided mechanism into symmetric 3x3 matrix form.

The source mechanism may be provided in different forms:

– as 3x3 matrix - symmetry is checked - one basis system has to be chosen, or NED as default is taken – as 3-element tuple or array - interpreted as strike, dip, slip-rake angles in degree – as 4-element tuple or array - interpreted as strike, dip, slip-rake angles in degree + seismic scalar moment in Nm – as 6-element tuple or array - interpreted as the 6 independent entries of the moment tensor – as 7-element tuple or array - interpreted as the 6 independent entries of the moment tensor + seismic scalar moment in Nm – as 9-element tuple or array - interpreted as the 9 entries of the moment tensor - checked for symmetry – as a nesting of one of the upper types (e.g. a list of n-tuples) - first element of outer nesting is taken

void cmopad_uniqueEuler ( double *  alpha,
double *  beta,
double *  gamma 
)

Uniquify euler angle triplet.

Puts euler angles into ranges compatible with (dip,strike,-rake) in seismology:

Parameters
[in,out]alphadip angle to put into range [0, pi/2]
[in,out]betastrike angle to put into range [0, 2*pi)
[in,out]gamma-rake angle to put into range [-pi, pi)
Note
If alpha is near to zero, beta is replaced by beta+gamma and gamma is set to zero, to prevent that additional ambiguity.

If alpha is near to pi/2, beta is put into the range [0,pi).
