C interface function to initialize the ActiveMQ shakeAlert decision module message listener. This function must be called first.

Parameters
[in]AMQuserAuthenticating username.
[in]AMQpasswordAuthenticating password.
[in]AMQdestinationQueue or topic name on the broker.
[in]AMQhostnameURL of host computer (e.g. computer.abc.def.edu).
[in]portPort number which is accepting connections on host computer.
[in]msReconnectNumber of milliseconds to wait for a reconnect attempt. If 0 or if maxAttempts is 0 then this command will be ignored.
[in]maxAttemptsNumber of attempts to connect before giving up. If 0 this command will be ignored.
[in]useTopicIf true then the message receiver connects to a topic (default). If false then the message receiver connects to a queue.
[in]clientAckIf true then the session will acknowledge a message has been received. If false then the session automatically acknowledges a client's receipt of a message either when the session has successfully returned from a call to receive or when the session's message listener has successfully processed the message (default). If true then the session is transacted and the acknowledgement of messages is handled internally.
[in]luseListenerIf false then you must actively check your messages (default). If true then use a message listener - this is useful when the driving program language is threaded and/or uses triggers.
[in]verbosecontrols verobosity.
Returns
0 indicates success.
